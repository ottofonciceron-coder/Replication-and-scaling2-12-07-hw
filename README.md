### Задание 1 Опишите основные преимущества использования масштабирования методами:
`активный master-сервер и пассивный репликационный slave-сервер;
master-сервер и несколько slave-серверов;
активный сервер со специальным механизмом репликации — distributed replicated block device (DRBD);
SAN-кластер.`

### 1.1 Активный master-сервер и пассивный репликационный slave-сервер(Приемущества)
1. `Отказоустойчивость - при падении master можно переключиться на slave`
2. `Резервное копирование - slave используется для бэкапов без нагрузки на master`
3. `Простота настройки - минимальная конфигурация для репликации`
4. `Разгрузка чтения - slave может обрабатывать SELECT-запросы (при необходимости)`

### 1.2 Master-сервер и несколько slave-серверов(Приемущества)

1. `Балансировка нагрузки чтения - запросы SELECT распределяются между несколькими slave`
2. `Горизонтальное масштабирование - можно добавлять новые slave по мере роста нагрузки`
3. `Высокая доступность - при падении одного slave остальные продолжают работу`
4. `Геораспределение - slave можно размещать в разных дата-центрах для ускорения локальных запросов`
5. `Специализация slave - один slave для отчётов, другой для поиска, третий для бэкапов`

### 1.3 Активный сервер со специальным механизмом репликации - DRBD (Distributed Replicated Block Device)

1. `Синхронная репликация на уровне блочного устройства - данные записываются одновременно на оба узла`
2. `Минимальные потери данных - при падении master данные уже есть на slave (RPO = 0)`
3. `Прозрачность для приложений - работа с DRBD как с обычным диском`
4. `Автоматическое переключение - возможно с Pacemaker/Corosync`
5. `Подходит для систем с высокими требованиями к целостности данных (финансы, биллинг)`


### 1.4 SAN-кластер (Storage Area Network)

1. `Общее хранилище для всех серверов - все узлы кластера работают с одними данными`
2. `Актив-актив режим - несколько серверов одновременно обрабатывают запросы на запись`
3. `Высокая производительность - специализированное оборудование с высокой пропускной способностью`
4. `Централизованное управление - резервное копирование, снапшоты, расширение на уровне SAN`
5. `Горизонтальное масштабирование - добавление новых серверов без репликации данных`
6. `Отказоустойчивость на уровне железа - RAID, дублированные контроллеры, пути доступа`
